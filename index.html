    • Jobs
      • Web and mobile era
      • This part of my career started around 2008 and continues today.
        • Spotify
        • I currently work at Spotify. I previously worked on a team that built music recommenders for users (in forms like the This Is playlists — greatest hits playlists for artists that stay fresh by updating daily based on listening patterns) — and Time Capsule), and I now work on a team that is building voice experiences.

          Some things I've worked on:

          • An image generator that composes thousands of playlist covers each day based on playlist contents
          • Internal apps (web apps, Chrome extensions, Electron apps) for evaluating and adjusting algorithmically-generated content
          • Interaction and content prototypes
          • Voice experience prototypes built in Alexa Skills and the browser (using the Web Speech API and DialogFlow)
          • External promotional web sites
          • Gathering data from users via surveys

          To do these things, I use full-stack web engineering, navigate a very unique internal infrastructure, and do ad hoc data digging. There is quite a bit of building things purely to see how users will react (in in-person user tests and A/B tests). I've learned much about the value and costs of gathering information before you commit to something.

        • PayPal
        • I worked at PayPal on the Shop team.
          The Shop site served coupons from various PayPal partners to hundreds of thousands of people per day. The front end was a single-page app built in AngularJS. The back end was a cluster of NodeJS servers that talked to PayPal services to manage the coupon and user information.
          We used TDD, pair programming, and GitFlow heavily. We prioritized the mobile web experience, which I've taken to heart ever since. And, perhaps less effectively, Scrum.
        • NPR
        • I worked at NPR on a station management app. It was called Composer, and it was written in Node and Backbone. Over 200 NPR member stations used it to add program schedules to their web sites and keep track of what they played so they could pay for it.

          I added this app's first automated tests (in order to safely do a big refactor to add OAuth to the API) and learned quite a bit about TDD as a result. Our team used a Kanban process and an unusually high level of design-development integration.

        • Modo Labs

        • The company built mobile apps for big institutions, mostly universities. It also wrapped up customers' existing data sources to make them easily mobile-consumable.

          I worked on iOS apps, with some web service development here and there.

        • Voter Activation Network
        • Their business was centered around a massive data warehousing web application. I helped with maintenance on that and also built a WYSIWYG form editor for the web. Later, I worked on back and front ends for their canvassing iOS app, MiniVAN Touch.
        • Obama for America
        • I spent a month as a "Data Fellow" during the 2008 presidential campaign. We automated manual tasks and wrote programs to look through voter contact history to find good targets for calls. 

          Indiana went blue, which it had not done since 1964. I met great people, and I saw how the relatively simple application of technology can make a big impact when applied to the right context.

    • Perspective
    • Here are my main principles for building software:

      • Write tests. Write enough to easily tell if something is broken when you change it but not so much that the tests become a burden unto themselves. Change program code to be more testable instead of making tests extremely elaborate.
      • Try to stick to code and tools you understand, even in third-party libraries. Sometimes, that can't be helped, but if you don't understand the code while you're building the feature, you will likely be forced to understand that code later—under pressure—while debugging a bug that made it to production.
      • Make sure the code you're considering writing is actually necessary.
